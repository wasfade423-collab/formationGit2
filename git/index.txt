Un commit est comme un point de sauvegarde dans votre projet. Un commit enregistre un instantanné de vos fichiers avec un message de modification d'illustration précis.
La commande git commit -m "message" valide l'ensemble des modifications effectuées sur tous les fichiers préparés.
La commande git commit -a -m "message" Valide l'ensemble des modifications effectuées su tousles fichiers du projet. Mais elle valide unique ment les fichiers déjà suivis(déjà préparer une fois. Si vous creer un fichier nouvellement avec git commit -a -m "message" le nouveau fichier n'est sauvegarder.)
La commande git reset --soft HEAD~1 permet d'annuler le dernier commit
La commande git log permet de voir la liste des commits
La commande git log --stat permet de voir quels fichiers ont été validés(modifiés dans .git) à chaque commit

Un workflow git c'est une recommandation sur la façon d'utiliser git pour accomplir un travail de façon cohérente et productive. 
Git utilise un flux de travail distribué qui vous permet de travailler sur votre code, de préparer les modifications et de les enregistrer dans votre dépôt local avant de les partager avec d'autres.
    Exemple de Workflow: centralisé, de branche de fonctionnalité, Gitflow, de duplication

git restore <file>          Annuler les modifications dans votre répertoire de travail (avant la mise en scène).
git restore --staged <file> Retirer un fichier de la zone de préparation (le déplacer hors de la zone de préparation).
git reset HEAD~             Annuler votre dernière validation (conserve les modifications dans votre répertoire de travail).
git commit --amend          Modifiez le dernier message de commit ou ajoutez des fichiers à votre dernier commit.
