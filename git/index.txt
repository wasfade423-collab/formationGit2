Un commit est comme un point de sauvegarde dans votre projet. Un commit enregistre un instantanné de vos fichiers avec un message de modification d'illustration précis.
La commande git commit -m "message" valide l'ensemble des modifications effectuées sur tous les fichiers préparés.
La commande git commit -a -m "message" Valide l'ensemble des modifications effectuées su tousles fichiers du projet. Mais elle valide unique ment les fichiers déjà suivis(déjà préparer une fois. Si vous creer un fichier nouvellement avec git commit -a -m "message" le nouveau fichier n'est sauvegarder.)
La commande git reset --soft HEAD~1 permet d'annuler le dernier commit
La commande git log permet de voir la liste des commits
La commande git log --stat permet de voir quels fichiers ont été validés(modifiés dans .git) à chaque commit


La commande git tag <nomTag> permet de creer un tagname (etiquette) légère pour marquer les points importants dans l'historique de notre projet
Pour créer une étiquette annotée(qui est recommandée) on utilise git tag -a <nomTag> -m "message" 
La commade git tag <nomTage> "codeHexDuCommit" permet de donner une étiquette à un commit existant
la commande git tag permet de voir la liste des etiquettes déjà donnée
La commande git show <nomTage> permet de voir les infos sur l'etiquettenomTage
Pour ajouter une étiquette à votre dépot distant on a git pus origin <nomTage>
Pour envoyer plusieurs tags on utilise git push --tags


La commande git stash -u est utilisée pour sauvegarder le travail d'une branch avant de switcher vers une autre.
La pile de stockage est une suite d'ensemble de modifications auxquelles s'ajoutent les modifications à chaque execution de git stash
